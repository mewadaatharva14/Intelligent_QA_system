What the Project Consists Of

The Intelligent PDF QA System is designed to enable dynamic, accurate question answering from user-uploaded PDF documents. When a PDF is uploaded, it undergoes a multi-step transformation pipeline. First, the system extracts textual data from the document. This raw text is then segmented into manageable chunks using LangChain, ensuring each piece retains semantic meaning and contextual relevance.

To prepare the text for processing by our language model, each chunk is converted into numerical form using a BERT tokenizer. These tokenized inputs are passed through a pre-trained deep learning model implemented using PyTorch, which computes logits representing the model’s interpretation of the text. The system then uses this representation to generate high-confidence answers to user questions — answers rooted in the actual content of the PDF, even for long or technical documents.

Limitations

  • Context Fragmentation: While the document is split into chunks for better handling, this can sometimes lead to answers missing the broader context, especially if critical information spans across multiple sections.
  • Quality of PDF Text Extraction: If the PDF is poorly formatted, scanned, or contains a lot of images, the text extraction phase may result in incomplete or inaccurate data, affecting downstream results.
  • Inability to Interpret Visuals: Diagrams, charts, and images embedded in PDFs are currently ignored. As a result, any important content conveyed visually will not be considered in the answer.
  • Answer Accuracy Relies on Training: The performance of the underlying model depends heavily on its training data. If the question falls outside the model’s understanding or the document doesn’t contain a direct answer, the system might generate misleading outputs.
  • Performance and Latency: Depending on the size of the PDF and server capacity, processing may take time. For very large PDFs, processing speed and memory usage can become limiting factors.
